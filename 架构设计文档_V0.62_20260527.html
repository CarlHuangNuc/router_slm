1. 项目概述

1.1 产品背景

VisionBuds 是一款AI智能蓝牙耳机产品。硬件端（蓝牙耳机）由ODM厂商万魔负责，本文聚焦手机端本地语音交互控制的软件架构设计。

1.2 核心定位

本地语音交互系统使用小模型流水线（VAD + ASR + Router + SLM + Action）实现两大本地任务：

  • 打开任意Android应用 — 通过语音快速启动手机App
  • 关键词检测与执行 — 识别预定义关键词并执行对应操作

1.3 系统边界

端职责
耳机端（上行）语音采集 + 唤醒词检测
耳机端（下行）仅执行"耳机拍照"一个关键词
手机端VAD检测 + ASR转写 + Router分流 + SLM推理 + Action执行
云端处理Router判定为非本地的复杂请求

2.3 模块封装策略

关键设计决策：
- VAD模块单独封装，供Agent流式调用
- ASR/Router/SLM/Action 封装成一个Android Service（本地语音引擎）
- Action模块统一通过发送系统Intent完成操作控制
- 整体由外部Agent控制器统一调度

3.3 Router模块

指标项要求
功能判定用户意图属于本地执行还是云端处理
模型大小< 0.3B (目标<0.2B)
模型架构Transformer-based
端到端准确率(含ASR)> 90%，本地精确率 > 95%
纯文本准确率> 93%，本地精确率 > 97%
Input Token< 300
Output Token< 5
时间延迟(Qualcomm)< 0.2s
OwnerTiaonan Duan

Router分流规则：

  • 打开Android App类意图 → 路由到本地
  • 操控耳机的关键词 → 路由到本地
  • 其他所有请求 → 路由到云端

3.4 SLM模块

指标项要求
功能本地意图理解，通过Function Call驱动Action
模型大小≤ 0.8B
语言支持中英文混杂
核心能力支持Function Call
Input Tokens< 200 (system + function call)
Output Tokens< 15
时间延迟≤ 0.2s
OwnerChenshi + Zhenghua Gu + Zhangwei

3.5 Action模块

指标项要求
功能接收SLM输出的Function Call，通过发送系统Intent执行具体操作
执行方式统一通过Android系统Intent指令下发
执行类型打开App / 关键词执行 / 耳机指令下发
失败策略优雅的提示用户错误，不传送到云端进行补救
OwnerChenshi + Junhao Li

6.2 Agent调用接口设计

Agent控制器对外暴露两类接口：

VAD模块接口（流式）：

接口说明
startVAD(audioStream)开始流式VAD检测
onSpeechStart(callback)语音开始回调
onSpeechEnd(callback)语音结束回调
stopVAD()停止VAD检测

本地语音引擎Service接口：

接口说明
processUtterance(audioData)处理一次完整语音(ASR→Router→SLM→Action)
onResult(callback)执行结果回调(local/cloud/action_result)
setHotwords(list)设置ASR热词列表
getEngineStatus()获取引擎运行状态
